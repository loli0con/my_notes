## 笔记
* ooad的目的：编写可维护、可复用、可拓展、灵活性好的程序
* ooad的手段：通过封装、继承、多态把程序的耦合度降低
* 不是类越多越好，类的划分是为了封装，但分类的基础的抽象，具有相同属性和功能的对象的抽象集合才是类
